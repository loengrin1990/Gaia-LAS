# Gaia review guide

Дата выгрузки: 2026-06-30.

## Что это

Gaia Local Analytics - локальная аналитическая оболочка вокруг проектной памяти
Obsidian. Система помогает выбрать проектный контекст, замаскировать ПД,
собрать безопасный prompt для внешнего анализа или отправить задачу в локальный
fallback через LM Studio.

Ключевая продуктовая граница: персональные данные не должны уходить за пределы
локальной машины. Внешний маршрут разрешается только после Veil-маскирования и
ручного подтверждения пользователя.

## Быстрый порядок знакомства

1. Прочитать `README.md` - общий продуктовый контекст, запуск, UX и проверки.
2. Открыть `docs/Архитектура.md` и `docs/Карта кода.md` - обзор слоев и файлов.
3. Посмотреть `docs/API.md` - HTTP-контракт локального сервера.
4. Пройтись по `docs/Модули/00 Карта модулей.md`, затем по нужным модулям.
5. Смотреть код с `app.py`, `gaia/server.py`, `gaia/projects.py`,
   `gaia/orchestrator.py`, `gaia/masking.py`, `gaia/memory.py`,
   `gaia/packaging.py`.
6. Запустить `python3 -m unittest discover -s tests`.

## Запуск

```bash
cd Local_Analytics_System
python3 -m gaia.config
python3 app.py
```

После запуска интерфейс доступен по адресу из конфига, по умолчанию:

```text
http://127.0.0.1:8787
```

Для запуска вне исходной машины нужно адаптировать `config.json`. В архиве есть
`config.review.example.json` с теми же секциями и переносимыми placeholder'ами.

## Что не включено в ревью-архив

- `runs/` с пользовательскими загрузками и рабочими артефактами запусков.
- журналы запросов, audit-файлы и черновики обновления памяти из Obsidian;
  они могут содержать рабочий контекст проектов.
- `.DS_Store`, `__pycache__`, `*.pyc`, временные lock-файлы Office.

## На что стоит обратить внимание ревьюеру

- Безопасность маршрутизации: `gaia/orchestrator.py`, `gaia/masking.py`,
  `gaia/policy.py`, `gaia/scribe.py`.
- Контракт API и обработка ошибок: `gaia/server.py`, тесты `tests/`.
- Управление проектами и группами памяти: `gaia/projects.py`,
  `tests/test_project_registry.py`.
- Работа с файловыми загрузками: `gaia/extraction.py`, `gaia/jobs.py`.
- Индексация Obsidian-памяти, наследование группового контекста и исключение
  сервисной документации из проектного контекста: `gaia/memory.py`,
  `gaia/config.py`.
- Retention и границы очистки: `gaia/archive.py`.
- UI как рабочая консоль, а не отдельный frontend-проект: `gaia/ui.py`.

## Текущий статус

MVP локального сервиса собран как stdlib-first Python-приложение. В коде есть
юнит-тесты на masking, профили, jobs, локальный статус, project registry,
memory index, Scribe, prompt rebuild, retention и UI review-сценарии.
