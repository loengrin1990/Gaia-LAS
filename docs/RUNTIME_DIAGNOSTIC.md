## Карта штатного запуска

Оба ярлыка рабочего стола (`Запустить аналитическую систему.command` и `Запустить Gaia.command`) совпадают: переходят в корень Gaia, проверяют конфигурацию и запускают `python3 -B app.py`. `app.py` импортирует `gaia.server.main` и запускает сервер с `open_window=True`. Отдельное окно вызывает тот же файл `gaia/gaia_window.js` с URL `http://127.0.0.1:8787`.

## Текущий процесс 8787

На момент диагностики порт `127.0.0.1:8787` свободен: Python-сервера Gaia нет. Поэтому нельзя получить новый HTTP-ответ `8787`, его frontend или фактический git commit работающего backend.

При этом найден оставшийся процесс отдельного окна:

- PID `69304`, запущен 2026-07-22 21:45:51, PPID `1`;
- команда: `/usr/bin/osascript -l JavaScript /Users/ilia/Documents/Gaia/gaia/gaia_window.js http://127.0.0.1:8787`;
- рабочий каталог: `/Users/ilia/Documents/Gaia`;
- этот процесс не слушает HTTP-порт.

Следовательно, отдельное окно пережило свой сервер и является независимым, осиротевшим runtime-объектом. Оно могло сохранять ранее загруженный DOM/WebKit-состояние, но уже не доказывает, что его backend соответствует HEAD. Это первая доказанная точка расхождения между видимым окном и проверяемым сервером: для окна на `8787` не существует обслуживающего процесса Gaia.

Вторая копия Gaia вне текущего репозитория среди доступных каталогов запуска не обнаружена. Исторические копии находятся только внутри `backups/` текущего репозитория и не являются целями ярлыков.

## Frontend и кэш

HTML не является отдельным bundle: `gaia/ui.py` читает `gaia/static/index.html` при импорте, а `Handler.do_GET` выдаёт уже загруженную строку. Скрипты и стили встроены в этот HTML. Отдельное окно передаёт только URL и не содержит копию frontend.

В коде не найдены service worker, Cache API, `localStorage`/`sessionStorage`-версии, второй каталог web-assets или query-параметры версии. HTTP-ответы отдают `Cache-Control: no-store, max-age=0` и `Pragma: no-cache`. Это исключает HTTP-кэш как объяснение для нового запроса к живому серверу, но не отменяет сохранённый DOM уже открытого WKWebView.

## Контрольный экземпляр 8788

Для диагностики был запущен тот же `python3 -B app.py` из HEAD на `127.0.0.1:8788` с отдельной временной конфигурацией и синтетическим storage. Внешние рабочие пространства и пользовательские данные не использовались.

- GET `/` вернул `200`, `Cache-Control: no-store, max-age=0`.
- SHA-256 отданного HTML совпал с `gaia/static/index.html`: `3d899b50609d9cb2d0a6e7d30d7ee5737e7c891800b23e722c693988d8db943f`.
- POST `/api/analyze` вернул `202`, review — `not_started`.
- Ранний POST `confirm` вернул `400`.
- POST `check` вернул `200`, затем GET review — `200`; при недоступной в тестовой конфигурации модели состояние корректно стало `review_error`, а не бесконечным ожиданием.
- POST `compile` до подтверждения вернул `400 material_not_confirmed`.

Успешная ветвь `confirm → compile` на 8788 не запускалась: временная конфигурация намеренно не назначала доступную модель, а менять runtime поддельным клиентом в штатном приложении означало бы менять диагностируемый путь. Её покрывает существующий HTTP E2E с временным storage и поддельными локальными моделями.

## Вывод и следующий адресный шаг

Основная подтверждённая причина текущего расхождения — осиротевшее отдельное окно, указывающее на `8787`, при отсутствии процесса на этом порту. Это не позволяет связать наблюдаемое в нём поведение с `c37eea9`; старые логи `202` не принадлежат текущему контрольному экземпляру HEAD.

Следующее исправление не требуется для доказательства, но для устойчивой ручной проверки нужен отдельный согласованный шаг по жизненному циклу launcher/window: явно распознавать отсутствие сервера и не оставлять окно без привязки к успешному старту. Оценка: 3–5% недельного лимита. До такого решения остаётся неизвестным только исторический процесс, обслуживавший старые `202`: на момент диагностики он уже не существует.

Временные конфигурация, storage, HTTP-след и контрольный процесс 8788 удаляются после фиксации этого документа. Продуктовый код, конфигурация пользователя, реальные данные, миграции, внешний доступ и Git push не изменялись.
