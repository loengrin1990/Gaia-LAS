# Детерминированный защитный контур

Новые файлы проходят путь: контролируемый приём → источник → извлечение → локальная детерминированная очистка → очищенное представление → временный адаптер → существующий сборщик пакета. Адаптер получает только очищенный текст; исходный текст и таблица псевдонимов ему недоступны.

Обязательные проверки: электронная почта, телефоны, URL и сетевые адреса, секреты и локальные словари рабочего пространства. Дополнительные правила включают документы, реквизиты и явные адреса. Совпадения адресов и словарные сущности помечаются как требующие проверки. Ошибка обязательной проверки блокирует передачу; ошибка дополнительной сохраняет результат со статусом, требующим проверки. Экспорт всегда запрещён (`export_allowed=false`).

Псевдонимы хранятся только в отдельной зоне соответствующего рабочего пространства. Они устойчивы внутри пространства и изолированы между пространствами. Безопасный отчёт содержит категории, количества, псевдонимы, безопасную позицию, статус, уверенность и версию правила — без исходных значений, текста, путей и имён файлов.

Правила имеют версию. Повторная очистка управляемого результата создаёт новую версию очищенного представления, не импортируя источник повторно; прежняя версия сохраняется. Повторная загрузка того же файла блокируется до временного адаптера. Локальные словари находятся в закрытой зоне соответствующего пространства и не выдаются API.

Этап 3.1 не включает локальную языковую модель, ручную проверку, автоматическое разрешение экспорта, этап 3.2 или этап 4.

## Локальная дополнительная проверка (этап 3.2)

После детерминированной очистки только её текст передаётся существующей локальной модели через loopback-маршрут. Ответ принимается только как компактный JSON с разрешённой категорией, координатами внутри очищенного текста, уровнем уверенности и безопасным кодом основания. Проверка запускается явно и синхронно: до запуска версия имеет состояние «Проверка ещё не запускалась», затем сразу переходит либо в «Требуется проверка», либо в «Готово к подтверждению», либо в безопасное состояние ошибки. Некорректный, слишком большой или недоступный ответ не разрешает подтверждение и не запускает анализ.

Закрытый локальный API проверки возвращает очищенный текст и координаты только своему рабочему пространству; безопасный API отчёта по-прежнему не возвращает текст. Пользователь может принять замену или оставить совпадение, затем подтверждает актуальную версию. Только такое подтверждение разрешает постановку существующего анализа. Подтверждение старой версии недействительно после новой очистки. Экспорт остаётся запрещённым.

Пользователь также может изменить категорию кандидата или добавить его координатный фрагмент в словарь своего пространства. Добавление обновляет локальный словарь и запускает новую очистку управляемого результата без повторной загрузки источника.

Когда пользовательское действие создаёт новую очищенную версию, Gaia создаёт для неё новое состояние проверки до возврата успешного ответа. В нём сохраняется история решений прежней версии, но не её подтверждение; локальная проверка выполняется заново для актуального очищенного текста. Это не допускает расхождения, при котором список материалов уже указывает на новую версию, а проверка ещё отсутствует.

Каждая находка привязана к конкретной версии очищенного текста, её координатам и одностороннему отпечатку ожидаемого фрагмента. Повторное решение по находке, устаревшие координаты и совпадение с уже созданным псевдонимом отклоняются без изменения текста. Повторная очистка сначала строит все непересекающиеся замены по неизменному исходному извлечению, затем применяет их с конца строки: созданный псевдоним не может стать входом следующего правила.

Словарное значение применяется только как целый самостоятельный фрагмент, а не как часть обычного слова, названия или номера. Локальная модель также не может предложить часть слова либо диапазон, пересекающий сформированный псевдоним. После каждого сохранённого решения Gaia показывает число оставшихся находок; когда их нет, версия готова к подтверждению. Решения разрешены только для завершённой проверки с необработанными находками; подтверждение — только для актуальной версии в состоянии «Готово к подтверждению».

Подтверждённая актуальная очищенная версия может быть передана только локальному [компилятору проектного контекста](CONTEXT_COMPILER.md); экспорт по-прежнему запрещён.

Пользовательский переход от добавления к проверке описан в [интерфейсе Gaia](USER_INTERFACE.md). Экран не раскрывает исходный источник, контрольные суммы или таблицу псевдонимов.
