# Gaia Local Analytics

Обновлено: 2026-07-03

Gaia Local Analytics - локальная сервисная оболочка для аналитической работы с проектной памятью Obsidian, файлами, аудио/видео, маскированием ПД, LM Studio и безопасной подготовкой пакетов для Codex/ChatGPT.

Эта папка находится вне `Проекты`, потому что она является технической документацией сервиса, а не пространством проектной памяти. Gaia не должна предлагать эту папку в выпадающем списке контекста и не должна использовать ее как аналитическую память для пользовательских запросов.

## Текущий MVP

- Код: локальная папка `Local_Analytics_System`.
- Точка входа: `app.py`.
- Модули сервиса: `gaia/*.py`.
- Конфиг: `Local_Analytics_System/config.json`.
- Проверка config: `python3 -m gaia.config`.
- Запуск: `python3 app.py` или локальный launcher, если он настроен вне git.
- URL по умолчанию: `http://127.0.0.1:8787`.
- Интерфейс: браузерная рабочая панель Focus с навигацией `Диалог`,
  `Память`, `Проекты`, `Диагностика`.
- Очередь задач: `POST /api/analyze` создает job, результат доступен через `GET /api/jobs/<job_id>`.
- Project Registry: создание, редактирование, группировка, проверка и восстановление проектных пространств памяти.
- Группы проектов: надпроектный контекст в `Obsidian Vault/Контексты/Группы`.
- Профили задач: общий анализ, записка, риск-анализ, кандидаты в память, план действий.
- Lore: локальный индекс эффективного контекста, где групповой слой наследуется проектом; для важных ответов формируется `evidence_plan` с точечными подтверждениями из первичных источников.
- Project Dialogues: история диалогов по каждому проекту, продолжение сообщения с rolling summary и передачей свежего package в Scribe.
- Scribe: Inbox для файлов проекта, предложения в память и запись выбранных карточек внутри Gaia после review.
- Archive: рабочие журналы, safety audit и retention cleanup.
- UI-review: безопасный пакет показывается перед копированием; копирование для Codex требует явного checkbox-подтверждения.
- UX-словарь: названия подсистем `Scribe`, `Lore`, `Veil` сохраняются как
  метки, а рабочие кнопки называют действие пользователя: собрать контекст,
  пересобрать контекст, предложить записи, записать в память.

## Граница ответственности

Gaia не является заменой Obsidian. Obsidian хранит проектную память, групповые регламенты и сервисную документацию. Gaia управляет маршрутом запроса: читает выбранный эффективный контекст, обрабатывает файлы, маскирует ПД, готовит безопасный пакет или отправляет задачу в локальную LM Studio.

Групповой контекст задает общие регламенты, шаблоны, методики и ограничения для нескольких проектов. При конфликте проектная память имеет приоритет.

## Безопасность

- Исходные ПД не отправляются во внешний контур.
- Codex/ChatGPT можно использовать только после локального маскирования, просмотра безопасного пакета и ручного подтверждения в UI.
- Если маскирование невозможно или детектор видит риск ПД без замен, внешний маршрут блокируется.
- Для offline-режима и спорных ПД используется локальный маршрут Hearth/LM Studio.

## Навигация

- `Архитектура.md` - общий контур и поток данных.
- `Карта кода.md` - соответствие файлов `gaia/*.py` модулям сервиса.
- `API.md` - локальные HTTP endpoints MVP.
- `Модули/Project Registry.md` - управление проектами и группами.
- `Модули/Lore.md` - эффективный контекст и наследование группового слоя.
- `Модули/Project Dialogues.md` - проектные истории диалогов и связь со Scribe.
- `Модули/Scribe.md` - staged обновление памяти без внешних скриптов.
- `Памятки/Безопасный внешний анализ.md` - review-порядок перед Codex/ChatGPT.
- `Памятки/Жизненный цикл задачи.md` - статусы jobs и polling.
- `Памятки/Настройка и запуск.md` - config, launcher, проверка запуска.
- `Памятки/Финальная интеграционная проверка.md` - smoke checklist, ограничения и итоговая архитектура.
- `Модули/` - технические описания модулей.
- `Журнал запросов/` - реальные журналы пользовательских запусков Gaia.
