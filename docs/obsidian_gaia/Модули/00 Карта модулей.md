# Карта модулей

- Gaia - `gaia/orchestrator.py`, центральный сценарий обработки запроса.
- Focus - `gaia/ui.py`, браузерный рабочий интерфейс и панель управления проектами.
- Project Registry - `gaia/projects.py`, создание, редактирование, группировка, проверка и восстановление проектных пространств памяти.
- Veil - `gaia/masking.py`, локальное маскирование ПД.
- Echo - `gaia/transcription.py`, локальная транскрибация аудио/видео.
- Lore - `gaia/memory.py`, чтение эффективного контекста: групповой слой + проектная память Obsidian.
- Project Dialogues - `gaia/conversations.py`, проектная история диалогов, rolling summary и продолжение turn через обычный Lore/Forge pipeline.
- Scribe - `gaia/scribe.py`, staged plan/apply обновления памяти после review; markdown-черновик остается fallback/export-режимом.
- Scribe Inbox - `gaia/scribe_inbox.py`, поиск файлов проекта, preview и подготовка package для Scribe без внешних скриптов.
- Excel Preview - `gaia/excel_preview.py`, структурная нормализация `.xlsx` в markdown preview без зависимости от `openpyxl`.
- Forge - `gaia/packaging.py`, сборка безопасного аналитического пакета.
- Hearth - `gaia/local_llm.py`, локальный fallback через LM Studio; лимиты
  `local_llm.prompt_char_limit` и `local_llm.max_tokens` должны соответствовать
  фактическому `Context Length` загруженной модели.
- Archive - `gaia/archive.py`, журналирование запусков сервиса и retention.
- Policy - `gaia/policy.py`, правила маршрутизации и первичный PII detector.
- Intake - `gaia/extraction.py`, прием файлов, извлечение текста и подключение Excel preview для `.xlsx`.
- Server - `gaia/server.py`, HTTP API для анализа, jobs, проектов, групп, диалогов, Scribe Inbox и локальных действий.

Группы проектов хранятся вне `Проекты`, в `Obsidian Vault/Контексты/Группы`, поэтому Gaia не считает их самостоятельными проектами. При анализе Lore сначала подмешивает групповой контекст, затем проектную память; при конфликте проектная память имеет приоритет.
