# Исправление локальной проверки и выбора файла Gaia

Дата: 2026-07-23

## Первичная причина ошибки локальной проверки

Маршрут `veil_review` передавал Qwen в качестве единственного образца готовую находку с условными координатами `0` и `1`. Локальная модель `qwen3:8b` воспроизводила этот пример либо строила координаты без проверки фактической позиции. Строгая проверка Gaia правильно отклоняла такие находки, но записывала только общий код `local_check_failed`. Поэтому пользователь видел `review_error`, хотя локальный Ollama ответил успешно.

## Что изменено

- Инструкция локальной проверки теперь не содержит фиктивной находки. Она явно требует пустой список при отсутствии подтверждённых остаточных значений и запрещает считать находкой названия категорий, метки полей и псевдонимы Gaia вида `Категория-01`.
- Срок ожидания этого небольшого локального запроса увеличен с 8 до 20 секунд, чтобы Qwen успевал сформировать JSON без обрыва на рабочем Mac.
- Строгая схема и проверка координат сохранены. Допускается только один внешний Markdown-блок или единственный полный JSON-объект; частичный JSON не исправляется.
- Для ошибки проверки сохраняются безопасные диагностические поля: идентификатор трассировки, локальный provider и модель, маршрут, длительность, размеры запроса и ответа, HTTP-результат, причина завершения, счётчики токенов и наличие reasoning. Текст материала и полный ответ модели не сохраняются.
- Общий `local_check_failed` заменён точными кодами причины: недоступность, тайм-аут, ошибка запроса, пустой или усечённый ответ, неверный JSON или схема, непригодные находки, ошибка сохранения и внутренняя ошибка.
- Нативный выбор файла в окне macOS теперь использует надёжный модальный `NSOpenPanel`; делегат WebKit удерживается глобальной ссылкой на всё время работы приложения. Это исключает потерю callback и зависимость от `webView.window` при открытии панели.

## Проверка

Проведён фактический локальный smoke-тест через Ollama `qwen3:8b` на `http://127.0.0.1:11434/api/chat` в изолированном временном рабочем пространстве Gaia:

1. `POST /api/analyze` с синтетическим очищенным `.txt` вернул `202` и состояние `not_started`.
2. `POST /api/reviews/{artifact_id}/check` вернул `200` и `ready_for_confirmation`.
3. `GET /api/reviews/{artifact_id}` вернул `200`, то же состояние, пустой список находок и пустой код ошибки.

Также успешно выполнены целевые тесты локальной проверки и launcher, проверка синтаксиса JXA, компиляция Python с временным bytecode-cache и полный набор тестов проекта.

## Границы изменения

Жизненный цикл запуска сервера, проверка готовности и логика окна из предыдущего исправления не менялись. Синтетические материалы были созданы только во временном каталоге и не затрагивали пользовательские проекты.
