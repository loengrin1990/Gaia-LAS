# Контракт хранения и безопасности Gaia

Этот документ — единый источник правил хранения. Он относится только к новым операциям; существующие рабочие пространства, журналы и диалоги не изменяются автоматически.

| Класс данных | Допустимое место | Срок и запреты |
|---|---|---|
| Исходные рабочие данные | Только закрытое рабочее пространство и оперативная память обработки | Не попадают в Git, MemoryHub, технические журналы, отчёты и экспорт. Временные загрузки удаляются после успешной операции; незавершённые — при включённой очистке не позднее заданного срока. |
| Очищенные рабочие данные | Локальный результат операции и память проекта | Перед внешней передачей проходят повторную проверку и ручное подтверждение. Не дублируются целиком в техническом журнале. |
| Технические метаданные | Локальные журналы Gaia | По умолчанию 30 дней для журналов и отчётов маскирования, до 180 дней для метаданных операций. Только статусы, количества, категории и безопасные идентификаторы. |
| История диалогов | Локальная сервисная документация Gaia | По умолчанию 90 дней при включённой очистке. Исходное сообщение не сохраняется: остаётся только безопасная техническая сводка. |
| Экспортные пакеты | Отдельный локальный каталог экспортов | Создаются только по явному действию пользователя, не попадают в журналы, Git или MemoryHub и не отправляются автоматически. |
| Данные разработки | Репозиторий Gaia и MemoryHub | Код, документация и синтетические тесты без рабочих данных. |

## Центральные настройки

Раздел `retention` локального `config.json` управляет сроками: `temporary_hours` (24), `journals_days` (30), `audit_days` (30), `conversations_days` (90), `operation_metadata_days` (180). Ноль означает, что автоматическая очистка этого класса отключена. Отрицательные значения запрещены проверкой конфигурации.

Очистка запускается только при явно включённом `cleanup_on_startup` либо отдельной команде обслуживания. Она не является миграцией существующих данных: перед массовой очисткой нужен отдельный предварительный просмотр и подтверждение пользователя.

## Журналы и отчёты

Журнал операции и отчёт маскирования содержат только технические сведения: безопасный идентификатор рабочего пространства, время, статус, типы файлов, категории и количество замен. В них запрещены сообщения, имена файлов, полный пакет, фрагменты документов, найденные значения и контекст вокруг них.

Локальные модели допускаются только на loopback-IP. Автоматической отправки во внешние сервисы нет.

## Git, MemoryHub и `output/`

`runs/`, `tmp/`, `outputs/`, `exports/`, резервные копии и локальная конфигурация исключены из Git. MemoryHub получает только разрешённые снимки кода, документации и синтетических тестов; загрузки, хранилища, резервные копии и базы исключены.

Каталог `output/pdf/` сейчас содержит версионируемые графические материалы документации. Он не меняется этим этапом: перед любым новым правилом для него требуется отдельно классифицировать создающий процесс и данные.

## Остающиеся ограничения

Результат операции существует в памяти запущенного процесса и может быть показан локальному пользователю. Миграция старых журналов и диалогов не запускалась; для неё нужен отдельный режим предварительного просмотра без чтения содержимого.
